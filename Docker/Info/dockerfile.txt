ARG - объявление переменых. Единственный параметр, который может находиться перед from

FROM <имя>:<тег> - базовый образ с которого сделать наш образ, dockerfile должен всегда начинаться с from

RUN - выполнить команду при сборке образа

WORKDIR - директория, в которой начинается работа контейнера, определяет рабочий каталог для инструкций RUN, CMD, ENTRYPOINT, COPY и ADD

COPY <место на локальной машине откуда копировать> <место в образе куда копировать> - копирование файлов

CMD ["", ""] - выполнить при старте контейнера

ENV <имя переменной> <значение переменной> - задать переменную окружения

EXPOSE <номер порта> - информация о том какие порты могут быть проброшены (проброс не происходит)

ENTRYPOINT ["", ""] - определяет выполняемый файл в контейнере, который должен запускаться по умолчанию в момент запуска контейнера

--------------------------------------------------------------

ADD - копирует файлы из указанного каталога в локальной системе или URL удаленной системы в указанный каталог внутри контейнера.
        В отличии от copy скачивает файлы по ссылке, разархивирует архивы

MAINTAINER - определяет автора информации в сгенерированных образах

USER - определяет имя или идентификатор пользователя для использования при запуске образа или в таких инструкциях, как RUN, CMD и ENTRYPOINT

VOLUME - определяет один или несколько общих томов в файловой системе хост-машины, которые будут доступны из контейнера
